# 성능 트렌드 분석 시스템 (Performance Trend Analysis) 상세 사양서

본 문서는 성능 트렌드 분석 대시보드의 디자인, 구성 요소, 데이터 로직 및 사용자 인터랙션을 상세히 정의합니다. 이 사양서를 통해 AI 소프트웨어 엔지니어가 동일한 기능을 가진 시스템을 재 구현할 수 있도록 하는 것이 목적입니다.

---

## 1. 프로젝트 개요 (Overview)
- **목적**: 1년 단위의 장기 성능 데이터를 시각화하고, 특정 기간의 성능 추이 및 분포를 다각도로 분석.
- **기술 스택**:
  - **Core**: HTML5, Vanilla JavaScript (ES6+), CSS3
  - **Visualization**: Chart.js (Main Chart), D3.js (Heatmaps, Hexbin) — CDN 로드
  - **Fonts**: Pretendard JP (CDN 로드)
  - **Build**: Vite (개발 서버, 번들링, API 프록시)
- **멀티 페이지 구조**: Vite MPA(Multi-Page Application) 방식으로 구성
  - `index.html` — JENNIFER Extension Labo 랜딩 페이지
  - `pta/index.html` — PTA 대시보드 (메인 앱)
  - `pta/help.html` — 분석 가이드 (팝업 도움말)

## 2. 디자인 시스템 (Design System)

### 2.1 색상 팔레트 (Color Palette)
현대적이고 프리미엄한 느낌을 위해 Glassmorphism과 세련된 색상을 사용합니다.
- **Primary (안정/安定)**: `#22c55e` (Green)
- **Warning (경계/警戒)**: `#facc15` (Yellow)
- **Danger (높음/高)**: `#ef4444` (Red)
- **Text**: Primary `#1e293b`, Secondary `#64748b`
- **Border**: `#e2e8f0`
- **Background**: `#ffffff` (Main), `#f8fafc` (Secondary)
- **월별 테마 색상**: 차트의 각 월 데이터는 12가지의 고정된 HSL 계열 색상을 순차적으로 사용하여 시각적 구분을 제공합니다.

### 2.2 타이포그래피 (Typography)
- **폰트**: `Pretendard JP` (Pretendard 기반, CDN 로드)
- **기본 크기**: `12px` (전체 가독성 향상)
- **컴포넌트 제목**: `0.80rem` ~ `1rem`, Semi-bold

### 2.3 레이아웃 구조 (Layout)
- **전체 레이아웃**: 사이드바를 제거하고 단일 컬럼 형태의 간결한 구조를 유지.
- **필터 영역**: 계층형 도메인 브레드크럼 셀렉터, 비지니스/인스턴스 전환 라디오 버튼, 비지니스 또는 인스턴스 드롭다운.
- **메인 차트 영역**: 7종 메트릭 토글, 이동평균(7일/30일) On/Off, 차트 타입(Line/Bar) 전환. X축 하단에는 **월별 통계(평균 수치 및 전월 대비 증감률)**가 상시(Persistent) 노출됨.
- **요약 카드 영역**: 카드의 높이와 폰트 크기를 컴팩트하게 줄여 1줄로 배치. 주요 지표 및 시스템 지표 포함하여 총 9개 카드 제공 (기간, 피크일, 평균응답시간, 평균TPS, 평균Hit수, 평균동시사용자, 평균에러율, 평균CPU, 평균힙메모리).
- **히트맵 그리드**: 2열 배치 (Overall 성능 분포, 요일x시간대별 부하 분포). 범례(Legend)는 직관성을 위해 한 줄(Single Line)로 간결하게 표시.
- **상세 슬라이드 패널 (Slide-in Detail Layer)**: 히트맵 셀 클릭 시 화면 좌측에서 나타나는 상세 데이터 확인용 오버레이 패널. 패널이 열려 있는 동안 메인 차트 인터랙션은 잠금 처리.

## 3. 데이터 아키텍처 (Data Architecture)

### 3.1 API 연동 및 데이터 처리
- **EndPoint**:
  - `/api/domain` — 도메인 목록 조회
  - `/api/instance` — 인스턴스 목록 조회
  - `/api/business` — 비지니스 목록 조회
  - `/api/dbmetrics/domain` — 도메인 전체 메트릭 조회
  - `/api/dbmetrics/instance` — 인스턴스별 메트릭 조회
  - `/api/dbmetrics/business` — 비지니스별 메트릭 조회
- **시간 패턴**: `yyyyMMddHH` 형식 (시간 단위 정밀 데이터 조회).
- **병합 로직**: API의 31일 조회 제한을 우회하기 위해 1년 데이터를 월 단위(12개 청크)로 분할 요청 후 병합, 시간순 정렬 및 중복 제거.
- **Mock 데이터**: API 호출 실패 시 시드 기반의 랜덤 데이터를 생성하여 시스템의 가용성 보장.

### 3.2 런타임 설정 (Runtime Configuration)
- **설정 파일**: `public/pta/config.js` (`window.PTA_CONFIG` 전역 객체, `BASE_URL`/`API_DOMAIN`만 포함)
- **빌드 독립적**: 빌드 후에도 `dist/pta/config.js`를 직접 수정하여 API 서버 주소 변경 가능.
- **참조 경로**: `src/main.js`가 `window.PTA_CONFIG`를 직접 읽어 `PTA_CFG.BASE_URL`, `PTA_CFG.API_DOMAIN`으로 사용 (별도 설정 모듈 파일 없음).
- **API 인증 토큰**: 클라이언트에는 존재하지 않음. `/api/` 요청이 업스트림으로 전달되는 시점에 리버스 프록시(운영: nginx, 로컬: Vite dev 서버)가 서버 측에서 주입 (§5.2 참고).

### 3.3 주요 메트릭 (Metrics)
| 메트릭 명         | API 키         | 단위  | 비고                                                         |
| :-------------- | :------------- | :---- | :--------------------------------------------------------- |
| 応答時間 (응답시간) | `service_time` | ms    | 성능 지표                                                    |
| TPS             | `service_rate` | TPS   | 처리량 지표                                                    |
| Hit数 (Hit수)    | `service_count` | Hits  | 호출 건수                                                    |
| 同時ユーザ数 (동시사용자) | `concurrent_user` | 人    | 세션 지표                                             |
| エラー率 (에러율)  | `err_rate`     | %     | `service_err_count / service_count × 100` 산출 (프론트엔드 자체 계산) |
| システムCPU      | `sys_cpu` / `max_sys_cpu` | %     | 인스턴스 전용                                        |
| ヒープメモリ使用率 | `heap_usage` | %     | 인스턴스 전용                                            |

## 4. 핵심 구성 요소 및 세부 로직

### 4.1 대상 선택 (도메인, 비지니스, 인스턴스)
- **계층형 도메인 선택기 (Breadcrumb)**:
  - API로부터 받아온 `groupHierarchy` 정보를 트리 구조로 변환하여 도메인 그룹 및 개별 도메인을 계층적으로 탐색.
  - 브레드크럼(Breadcrumb) 형태의 UI를 통해 현재 위치를 직관적으로 표현하며, 각 단계를 클릭하면 하위 항목이 팝오버로 표시됨.
  - 그룹 선택 시 해당 그룹 내 첫 번째 도메인을 자동 선택.
- **비지니스/인스턴스 전환 (Target Type Toggle)**:
  - 라디오 버튼으로 "인스턴스" 또는 "비지니스" 모드를 전환.
  - 비지니스 모드 시 비지니스 셀렉터가 표시되고, `businessIndex`에 기반한 들여쓰기(indent)로 계층 관계를 시각적으로 표현.
  - 인스턴스 모드 시 인스턴스 셀렉터가 표시되며, 개별 인스턴스 선택 시 CPU/메모리 메트릭 버튼이 활성화됨.
  - CPU/메모리 메트릭은 인스턴스 전용이므로, 비지니스 모드 또는 "전체 인스턴스" 선택 시 해당 요약 카드가 비활성화(disabled) 처리.
- **자동 데이터 갱신**: 도메인, 인스턴스, 비지니스 변경 시 별도의 조회 버튼 없이 모든 차트와 히트맵 데이터가 즉시 재로드됨.

### 4.2 메인 트렌드 차트
- **이동 평균선 (SMA)**: 7일 및 30일 이동평균 계산 루틴 포함.
- **동적 투명도**: 이동평균 On 시 배경 데이터 투명도 상승(강조), Off 시 원복.
- **CPU 콤보 차트**: 평균 CPU(Bar/Line)와 최대 CPU(항상 Red Line)를 중첩 표시.
- **월별 경계선**: 차트 내에서 월이 바뀌는 지점에 시각적 경계를 표시하고, 마우스 호버 시 해당 월 영역을 색상으로 하이라이트.
- **월별 하단 통계**: 차트 X축 아래에 각 월의 평균값과 전월 대비 증감률(%)을 HTML로 렌더링.

### 4.3 Overall 히트맵 (데이터 분포)
- **X축 (Hits)**: 현재 선택된 기간의 호출 건수 중앙값을 **동적으로 산출**하여 4분면 축의 기준으로 삼음.
- **Y축 (Performance)**: 1년 전체 데이터의 성능 지표 중앙값을 **최초 1회 산출 후 고정(캐싱)**하여 절대적인 성능 기준을 제공.
- **시각화**: D3 Hexbin을 사용하여 부하(X) 대비 성능(Y)의 밀집도를 표현. 밀도에 따른 녹색 계열 투명도 그라데이션(sqrt 곡선) 적용.
- **인터랙션**: 개별 Hexagon 클릭 시 해당 데이터 포인트들의 상세 목록이 좌측 슬라이드 패널로 표시됨.

### 4.4 요일 x 시간대별 분포 히트맵
- **기준 데이터**: 선택된 메트릭의 평균 수치. (단, 'Hit수' 선택 시에는 호출 건수 기준)
- **통계적 임계치 자동 계산**:
  - **安定 (안정)**: 데이터 <= Q3 (75% 지점)
  - **警戒 (경계)**: Q3 < 데이터 <= Q3 + (1.5 * IQR)
  - **高 (높음)**: 데이터 > Q3 + (1.5 * IQR) (통계적 이상치)
- **왜곡 방지**: 데이터가 0(Hits 0)인 구간은 평균 산출에서 제외.
- **인터랙션**: 개별 셀 클릭 시 해당 요일·시간대의 날짜별 상세 데이터가 좌측 슬라이드 패널로 표시됨. 인라인 가로 막대차트 포함.

### 4.5 오버레이 상세 슬라이드 패널 (Detail Slide Layer)
- 히트맵 셀/Hexagon 클릭 시, 상세 메트릭 수치 배열을 좌측 패널(Slide-in)로 열어서 확인 가능.
- 내부에 인라인 형태의 가로 막대차트(Bar Chart)를 포함하여 지표 간 수치 비교가 용이하며 시각적 피드백(Row Hover) 제공.
- 에러율, CPU 등 부가 지표를 종합해서 노출.

## 5. 빌드 및 배포 아키텍처

### 5.1 Vite MPA 빌드 설정
- `vite.config.js`의 `rollupOptions.input`에 3개의 entry point 등록: `main(index.html)`, `pta(pta/index.html)`, `help(pta/help.html)`.
- 에셋 출력 경로: `pta/asserts/` 하위에 JS/CSS 번들 생성.
- 빌드 후 `fix-paths.js`가 자동 실행되어 `dist/pta/index.html`, `dist/pta/help.html` 내의 `../pta/asserts/` 경로를 `./asserts/`로 보정.

### 5.2 개발 서버 프록시
- Vite 개발 서버(포트 `5177`)에서 `/api` 경로 요청을 `BASE_URL`로 프록시.
- `secure: false` 설정으로 자가서명 인증서 환경 지원.
- `configure` 훅에서 `proxy.on('proxyReq', ...)`을 통해, 업스트림으로 요청이 전달되기 직전에 `.env`의 `DEV_API_TOKEN` 값을 `token` 쿼리파라미터로 주입 (클라이언트 코드는 토큰을 모름).
- 같은 훅에서 `proxy.on('proxyRes', ...)`으로 CORS 관련 헤더(`Cross-Origin-Resource-Policy` 등) 주입.
- 운영 환경(nginx)에서는 `set $args "${args}&token=<token>";`로 동일한 역할을 수행 (`DEPLOY_GUIDE_NGINX.md` 참고).

## 6. 사용자 인터랙션 및 UX

- **기간 선택**: 차트 드래그 앤 셀렉트. 방향키(`←`, `→`)를 통한 선택 영역 이동. `Shift`+방향키를 통한 선택 영역 확장/축소 (활성 경계 자동 전환).
- **ESC 인터랙션**: 기간 선택 영역 즉시 해제 및 전체 데이터 원복. 해당 키 클릭 시 열려있는 상세 슬라이드 패널도 닫힘.
- **로딩 오버레이**: 글래스모피즘 효과와 스피너를 활용한 로딩 시각화 (`データロード中...`).
- **가독성 최적화**:
  - 기본 폰트 크기를 `12px`로 줄여 컴팩트한 화면 제공.
  - 툴팁 및 데이터 수치는 소수점 둘째 자리 고정 표기.
  - 히트맵 등 범례(Legend) 구역을 일렬(Single Line) 형태로 간결하게 배열.
- **포커스 유지**: 상세 데이터를 볼 때 화면 좌측에 슬라이드 패널 오버레이 시, 배경(메인 차트) 상호작용을 차단하여 사용자 집중 유도.
- **팝업 도움말 시스템**:
  - `pta/help.html`: 지표 정의 및 분석 가이드가 포함된 독립 페이지. "시스템 개요 > 메인 시계열 차트 > 히트맵 분석"의 구조.
  - `window.open`: 팝업 창 방식을 통해 대시보드와 도움말을 동시에 확인 가능.
  - **문맥 연동**: `mainHelpBtn`은 메인 차트 섹션으로, 히트맵별 `help-btn-small` 버튼은 해당 분석 섹션(`#overall`, `#dayhour`)으로 자동 연결.
  - **팝업 재사용**: 이미 열린 도움말 팝업이 있으면 해시만 업데이트하고 포커스 전환.
