# 성능 트렌드 분석 시스템 (Performance Trend Analysis) 상세 사양서

본 문서는 성능 트렌드 분석 대시보드의 디자인, 구성 요소, 데이터 로직 및 사용자 인터랙션을 상세히 정의합니다. 이 사양서를 통해 AI 소프트웨어 엔지니어가 동일한 기능을 가진 시스템을 재 구현할 수 있도록 하는 것이 목적입니다.

---

## 1. 프로젝트 개요 (Overview)
- **목적**: 1년 단위의 장기 성능 데이터를 시각화하고, 특정 기간의 성능 추이 및 분포를 다각도로 분석.
- **기술 스택**:
  - **Core**: HTML5, Vanilla JavaScript (ES6+), CSS3
  - **Visualization**: Chart.js (Main Chart), D3.js (Heatmaps, Hexbin)
  - **Fonts**: Pretendard JP (일본어/한국어 최적화 가독성 폰트)
  - **Build**: Vite (개발 서버 및 번들링)

---

## 2. 디자인 시스템 (Design System)

### 2.1 색상 팔레트 (Color Palette)
현대적이고 프리미엄한 느낌을 위해 Glassmorphism과 세련된 색상을 사용합니다.
- **Primary (안정/안정)**: `#22c55e` (Green)
- **Warning (경계/警戒)**: `#facc15` (Yellow)
- **Danger (높음/高)**: `#ef4444` (Red)
- **Text**: Primary `#1e293b`, Secondary `#64748b`
- **Border**: `#e2e8f0`
- **Background**: `#ffffff` (Main), `#f8fafc` (Secondary)
- **월별 테마 색상**: 차트의 각 월 데이터는 12가지의 고정된 HSL 계열 색상을 순차적으로 사용하여 시각적 구분을 제공합니다.

### 2.2 타이포그래피 (Typography)
- **폰트**: `Pretendard JP Variable` (Pretendard 기반)
- **기본 크기**: `12px` (전체 가독성 향상)
- **컴포넌트 제목**: `0.80rem` ~ `1rem`, Semi-bold

### 2.3 레이아웃 구조 (Layout)
- **필터 영역**: 도메인 및 인스턴스 선택 (인스턴스 미선택 시 도메인 전체 조회).
- **메인 차트 영역**: 6종 메트릭 토글, 이동평균(7일/30일) On/Off, 차트 타입(Line/Bar) 전환.
- **요약 카드 영역**: 8개 카드 (기간, 피크일, 주요 지표 평균 5종, 시스템 지표 2종).
- **히트맵 그리드**: 2열 배치 (Overall 성능 분포, 요일x시간대별 부하 분포).

## 3. 데이터 아키텍처 (Data Architecture)

### 3.1 API 연동 및 데이터 처리
- **EndPoint**: `/api/dbmetrics`, `/api/domain`, `/api/instance`
- **시간 패턴**: `yyyyMMddHH` 형식 (시간 단위 정밀 데이터 조회).
- **병합 로직**: API의 31일 조회 제한을 우회하기 위해 1년 데이터를 30일 단위로 분할 요청 후 병합 및 시간순 정렬.
- **Mock 데이터**: API 호출 실패 시 시드 기반의 랜덤 데이터를 생성하여 시스템의 가용성 보장.

### 3.2 주요 메트릭 (Metrics)
| 메트릭 명 | API 키 | 단위 | 비고 |
| :--- | :--- | :--- | :--- |
| 応答時間 (응답시간) | `service_time` | ms | 성능 지표 |
| TPS | `service_rate` | TPS | 처리량 지표 |
| Hit数 (Hit수) | `service_count` | Hits | 호출 건수 |
| 同時ユーザ数 (동시사용자) | `concurrent_user` | 人 | 세션 지표 |
| システムCPU | `sys_cpu` / `max_sys_cpu` | % | 인스턴스 전용 |
| ヒープメモリ使用率 | `heap_usage` | % | 인스턴스 전용 |

## 4. 핵심 구성 요소 및 세부 로직

### 4.1 메인 트렌드 차트
- **이동 평균선 (SMA)**: 7일 및 30일 이동평균 계산 루틴 포함.
- **동적 투명도**: 이동평균 On 시 배경 데이터 투명도 상승(강조), Off 시 원복.
- **CPU 콤보 차트**: 평균 CPU(Bar/Line)와 최대 CPU(항상 Red Line)를 중첩 표시.

### 4.2 Overall 히트맵 (데이터 분포)
- **X축 (Hits)**: 현재 선택된 기간의 호출 건수 중앙값을 **동적으로 산출**하여 4분면 축의 기준으로 삼음.
- **Y축 (Performance)**: 1년 전체 데이터의 성능 지표 중앙값을 **최초 1회 산출 후 고정(캐싱)**하여 절대적인 성능 기준을 제공.
- **시각화**: D3 Hexbin을 사용하여 부하(X) 대비 성능(Y)의 밀집도를 표현.

### 4.3 요일 x 시간대별 분포 히트맵
- **기준 데이터**: 선택된 메트릭의 평균 수치. (단, 'Hit수' 선택 시에는 호출 건수 기준)
- **통계적 임계치 자동 계산**:
  - **安定 (안정)**: 데이터 <= Q3 (75% 지점)
  - **警戒 (경계)**: Q3 < 데이터 <= Q3 + (1.5 * IQR)
  - **高 (높음)**: 데이터 > Q3 + (1.5 * IQR) (통계적 이상치)
- **왜곡 방지**: 데이터가 0(Hits 0)인 구간은 평균 산출에서 제외.

## 5. 사용자 인터랙션 및 UX

- **기간 선택**: 차트 드래그 앤 셀렉트. 방향키(`←`, `→`)를 통한 1개월 단위 영역 이동.
- **ESC 인터랙션**: 기간 선택 영역 즉시 해제 및 전체 데이터 원복.
- **로딩 오버레이**: 글래스모피즘 효과와 스피너를 활용한 로딩 시각화 (`数据ロード中...`).
- **가독성 최적화**: 툴팁 및 범례 수치 소수점 둘째 자리 고정 표기.
- **팝업 도움말 시스템**:
  - `help.html`: 지표 정의 및 분석 가이드가 포함된 독립 페이지. "시스템 개요 > 메인 시계열 차트 > 히트맵 분석"의 구조.
  - `window.open`: 팝업 창 방식을 통해 대시보드와 도움말을 동시에 확인 가능.
  - **문맥 연동**: `mainHelpBtn`은 메인 차트 섹션으로, 차트별 버튼은 해당 분석 섹션으로 자동 연결.
