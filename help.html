性能トレンド分析 ガイド

1. システム概要

性能トレンド分析ダッシュボードは、ウェブアプリケーションの長期的な性能推移を可視化し、システムの状態を多角的に分析するためのツールです。1年間のデータを1時間単位の精度で保持し、トレンドの把握やキャパシティプランニングに活用できます。

2. メイン時系列チャート

アプリケーション上部に配置されたメインチャートは、選択したメトリクスの時間経過に伴う変化を確認するための主要な指標です。

2.1 各指標（メトリクス）の定義

  • 応答時間 Response Time (ms): リクエストの処理完了までにかかった平均時間。ユーザー体験に直結する重要な指標です。
  • TPS Transactions Per Second: 1秒あたりの平均処理件数。システムの現在のスループットを示します。
  • Hit数 Hits (Count): 特定の時間枠内の総リクエスト数。システムの負荷量（トラフィック）を測定します。
  • 同時ユーザ数 Concurrent Users: システムにアクティブに接続しているユーザー数。
  • CPU / メモリ インスタンスごとの物理リソース使用率。平均値と最大値(Peak)を同時に分析することで、リソース不足を早期に検知できます。

2.2 ダッシュボードの操作方法

  • 期間の絞り込み: メインチャート上の特定の範囲をマウスでドラッグすると、その期間の統計データが全カードとヒートマップに即座に反映されます。
  • 範囲の移動: キーボードの方向キー (←, →) を押すと、選択した1ヶ月の範囲を前後に移動できます。
  • 選択解除: Esc キーを押すと、選択がリセットされ、1年全体のデータに戻ります。
  • 移動平均の活用: 「移動平均」をOnにすると、短期的な変動を抑えた長期的なトレンドが視覚化され、本質的な変化を捉えやすくなります。

3. Overall ヒットマップ分析 (相関分析)

このチャートは、負荷(Hits)と性能(Response Time)の相関を示します。データは中央値(Median)によって4つのエリアに分類されます。

左上: 効率不良エリア 低負荷にもかかわらず応答が遅い状態。非効率なクエリや外部接続の遅延が疑われます。
右上: 限界エリア 負荷が高まると同時に応答も遅くなっている状態。サーバーのリソース増設が必要なサインです。
左下: 理想エリア 負荷が低く、応答も非常に速い安定した状態。
右下: 高性能エリア 高い負荷がかかっているにもかかわらず、応答速度が維持されている非常に効率的な状態。

4. 曜日 x 時間帯別 分析 (パターン分析)

日次および週次の稼働パターンを把握するためのヒートマップです。統計的な標準偏差に基づき、自動的にステータスを分類します。

  • 安定 (Stable): 過去の統計から見て平均的な動作範囲。
  • 警戒 (Warning): 通常よりも平均負荷が高い、または応答が低下している時間帯。注意深い監視が必要です。
  • 高 (High): 明らかなピーク。リソースの限界に近い、または異常なアクセス集中が発生している可能性があります。
